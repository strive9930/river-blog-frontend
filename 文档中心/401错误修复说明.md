# 401 错误导致自动登出问题修复

## 🎯 问题现象

登录成功后，首页加载时出现以下错误：
```
GET https://localhost:7051/api/userfrontend/dashboard/stats 401 (Unauthorized)
Token: ❌ 不存在
```

然后页面自动跳转回登录页。

---

## 🔍 问题原因分析

### 根本原因

**API 请求地址配置错误**

从日志可以看到：
```
GET https://localhost:7051/api/userfrontend/dashboard/stats 401
```

但实际配置应该是：
- `.env`: `VITE_API_BASE_URL=http://localhost:5001/api`
- `vite.config.ts`: proxy target = `http://localhost:5000`

**配置不一致导致的问题：**
1. vite.config.ts 中的 proxy 配置指向 `http://localhost:5000`
2. 但后端实际运行在 `https://localhost:7051`
3. API 请求被发送到错误的地址
4. 返回 401 Unauthorized
5. 响应拦截器捕获 401 错误，清除 Token 并跳转到登录页

---

## ✅ 修复方案

### 修复 1：更新 vite.config.ts

**文件：** `vite.config.ts`

#### 变更前：
```typescript
server: {
  port: 3000,
  proxy: {
    '/api': {
      target: 'http://localhost:5000', // ❌ 错误的地址
      changeOrigin: true,
    },
  },
}
```

#### 变更后：
```typescript
server: {
  port: 3000,
  proxy: {
    '/api': {
      target: 'https://localhost:7051', // ✅ 正确的后端地址
      changeOrigin: true,
      secure: false, // ✅ 如果后端使用自签名证书
    },
  },
}
```

---

### 修复 2：更新 .env 文件

**文件：** `.env`

#### 变更前：
```env
VITE_API_BASE_URL=http://localhost:5001/api
```

#### 变更后：
```env
VITE_API_BASE_URL=https://localhost:7051/api
```

---

### 修复 3：优化 Home.vue 错误处理

**文件：** `src/views/Home.vue`

#### 变更前：
```typescript
try {
  await userStore.loadDashboardStats()
} catch (error) {
  console.error('加载仪表盘数据失败:', error)
}
```

#### 变更后：
```typescript
try {
  await userStore.loadDashboardStats()
} catch (error: any) {
  // 如果是 401 错误，不要显示错误信息（响应拦截器已经处理了）
  if (error.response?.status !== 401) {
    console.error('加载仪表盘数据失败:', error)
  }
}
```

**改进点：**
- ✅ 避免在 401 错误时重复输出错误日志
- ✅ 响应拦截器已经处理了 401 错误（清除 Token + 跳转）
- ✅ 添加 `isLoggedIn` 状态检查，便于调试

---

### 修复 4：修复 NavBar.vue 变量名错误

**文件：** `src/components/NavBar.vue`

#### 变更前：
```vue
<div 
  class="mobile-overlay" 
  v-if="!isSmallScreenHoverEnabled && showMobileMenu && mobileMenuOpen"
  @click="toggleMobileMenu"
></div>
```

#### 变更后：
```vue
<div 
  class="mobile-overlay" 
  v-if="!isSmallScreenHoverEnabled && showMobileMenuOnHover && mobileMenuOpen"
  @click="toggleMobileMenu"
></div>
```

**修复原因：**
- 模板中使用了 `showMobileMenu`，但实际定义的变量是 `showMobileMenuOnHover`
- 导致 Vue 警告：`Property "showMobileMenu" was accessed during render but is not defined on instance`

## 📝 配置说明

### 后端地址配置优先级

1. **vite.config.ts 中的 proxy**（开发环境）
   - 优先级最高
   - 所有 `/api` 开头的请求都会被代理
   - 适用于解决跨域问题

2. **.env 中的 VITE_API_BASE_URL**
   - 如果未配置 proxy，使用此地址
   - 适用于生产环境或直接请求

3. **api.ts 中的 baseURL**
   - 默认值：`import.meta.env.VITE_API_BASE_URL || 'http://localhost:5001/api'`
   - 作为后备配置

### 推荐配置

**开发环境：**
```typescript
// vite.config.ts
server: {
  proxy: {
    '/api': {
      target: 'https://localhost:7051',
      changeOrigin: true,
      secure: false,
    },
  },
}
```

```env
# .env
VITE_API_BASE_URL=https://localhost:7051/api
```

**生产环境：**
```env
# .env.production
VITE_API_BASE_URL=https://your-production-api.com/api
```
