# 前端权限系统快速参考

## 🚀 快速开始

### 1. 登录获取权限

```typescript
import { useUserStore } from '@/stores/user';

const userStore = useUserStore();

// 登录（一次性获取所有数据）
await userStore.login({
  email: 'user@example.com',
  password: 'password123'
});

// 登录后自动拥有：
// - userInfo (用户信息)
// - permissions (权限码列表)
// - menus (菜单树)
// - roles (角色列表)
```

## 🎯 权限指令

### v-permission - 按钮级权限控制

```vue
<template>
  <!-- 单个权限 -->
  <button v-permission="'article:create'">发布文章</button>
  
  <!-- 多个权限（满足任一即可） -->
  <button v-permission="['article:edit', 'article:delete']">
    编辑或删除
  </button>
</template>
```

**工作原理：**
- ✅ 有权限 → 元素正常显示
- ❌ 无权限 → 元素从 DOM 中移除

---

## 👑 角色指令

### v-role - 角色控制

```vue
<template>
  <!-- 单个角色 -->
  <div v-role="'Admin'">
    管理员专属内容
  </div>
  
  <!-- 多个角色（满足任一即可） -->
  <div v-role="['Admin', 'VIP']">
    管理员或 VIP 可见
  </div>
</template>
```

## 🎨 组合式函数

### usePermission - 权限检查

```typescript
import { usePermission } from '@/composables/usePermission';

const {
  hasPermission,    // 检查权限
  hasRole,         // 检查角色
  isAdmin,         // 是否管理员
  isVIP,          // 是否 VIP
  userInfo,       // 用户信息
  permissions,    // 权限码列表
  roles          // 角色列表
} = usePermission();

// 使用示例
if (hasPermission('article:create')) {
  // 执行创建操作
}

if (isAdmin.value) {
  // 管理员专属逻辑
}
```

### useMenu - 菜单操作

```typescript
import { useMenu } from '@/composables/usePermission';

const {
  mainMenus,      // 一级菜单
  getChildMenus,  // 获取子菜单
  findMenuByPath, // 根据路径查找菜单
  allMenus       // 所有菜单
} = useMenu();

// 使用示例
const firstLevelMenus = mainMenus.value;
const childMenus = getChildMenus('menu-id-1');
```

### useDashboard - 仪表盘数据

```typescript
import { useDashboard } from '@/composables/usePermission';

const {
  stats,          // 完整统计数据
  totalArticles,  // 文章总数
  totalComments,  // 评论总数
  totalViews,     // 总访问量
  loadStats       // 加载数据
} = useDashboard();

onMounted(async () => {
  await loadStats();
});
```

## ⚠️ 注意事项

### 1. 权限码命名规范

建议使用 `资源:操作` 的格式：

```typescript
'article:view'      // 查看文章
'article:create'    // 创建文章
'article:edit'      // 编辑文章
'article:delete'    // 删除文章
'comment:audit'     // 审核评论
'user:manage'       // 用户管理
```

### 2. 角色命名规范

使用英文驼峰或大写：

```typescript
'Admin'      // 管理员
'VIP'        // VIP 用户
'Editor'     // 编辑
'SuperAdmin' // 超级管理员
```

### 3. 性能优化

- ✅ 权限数据在登录时一次性加载
- ✅ 避免频繁请求权限接口
- ✅ 使用 computed 缓存权限判断结果

### 4. 安全性

- ⚠️ 前端权限控制仅为用户体验优化
- ⚠️ 不能替代后端验证
- ⚠️ 所有敏感操作必须在后端进行权限验证
- ⚠️ Token 过期时会自动清除权限数据并跳转登录页

## 🎯 最佳实践

1. **登录流程**
   - ✅ 使用 `userStore.login()` 一次性获取所有数据
   - ❌ 不要登录后再调用其他接口获取权限/菜单

2. **权限检查**
   - ✅ 使用 `v-permission` 指令进行按钮级控制
   - ✅ 使用 `hasPermission()` 进行逻辑判断
   - ❌ 不要硬编码权限检查逻辑

3. **角色检查**
   - ✅ 使用 `v-role` 指令进行角色控制
   - ✅ 使用 `isAdmin.value` 等计算属性
   - ❌ 不要直接访问 `userInfo.roles`

4. **错误处理**
   - ✅ 路由守卫自动处理 Token 失效
   - ✅ 401 错误自动跳转登录页
   - ❌ 不要忽略权限加载失败

## 🐛 常见问题

### Q1: 登录后看不到菜单？

**A:** 检查后端是否返回了 `menus` 字段：
```typescript
console.log('Menus:', response.data.menus);
```

### Q2: 权限判断不生效？

**A:** 确认权限码是否正确：
```typescript
console.log('Permissions:', userStore.permissions);
// 确保使用的权限码在列表中
```

### Q3: 刷新后丢失状态？

**A:** 检查 localStorage 是否有数据：
```javascript
console.log(localStorage.getItem('auth_token'));
console.log(localStorage.getItem('user_info'));
```
