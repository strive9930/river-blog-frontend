# 登录后立即返回登录页面问题修复

## 🎯 问题现象

登录成功后，页面立即跳转回登录页面，无法正常访问其他页面。

---

## 🔍 问题原因分析

### 根本原因

路由守卫的逻辑存在缺陷，导致以下问题：

1. **状态检查条件错误**
   - 原代码检查 `!userStore.permissions || userStore.permissions.length === 0`
   - 但登录时已经设置了 permissions，这个条件可能为 false
   - 导致跳过用户信息加载，但 Store 状态可能未正确初始化

2. **缺少 localStorage 恢复机制**
   - 页面刷新或重新导航时，Pinia Store 状态会重置
   - 原代码没有从 localStorage 恢复用户状态
   - 导致 `userStore.userInfo` 为 null

3. **错误的登出逻辑**
   - 当 `loadUserInfo()` 失败时，直接调用 `userStore.logout()`
   - 但如果只是网络问题或接口未实现，不应该清除登录状态

---

## ✅ 修复方案

### 修复 1：优化路由守卫逻辑

**文件：** `src/router/index.ts`

#### 变更前：
```typescript
// ❌ 错误的检查条件
const needLoadFullInfo = !userStore.permissions || userStore.permissions.length === 0;

if (needLoadFullInfo && token) {
  await userStore.loadUserInfo(); // 可能失败导致 logout
}
```

#### 变更后：
```typescript
// ✅ 正确的检查条件
const needLoadFullInfo = !userStore.userInfo;

// 1. 首先尝试从 localStorage 恢复
if (needLoadFullInfo && token) {
  const storedUserInfo = localStorage.getItem('user_info');
  if (storedUserInfo) {
    const userInfo = JSON.parse(storedUserInfo);
    userStore.userInfo = userInfo;
    userStore.permissions = userInfo.permissions || [];
    userStore.menus = userInfo.menus || [];
  }
}

// 2. 如果仍然没有用户信息，才从服务器加载
if (!userStore.userInfo && token) {
  try {
    await userStore.loadUserInfo();
  } catch (error) {
    userStore.logout();
    next('/login');
    return;
  }
}
```

**改进点：**
- ✅ 先尝试从 localStorage 恢复（快速、可靠）
- ✅ 只有在必要时才调用服务器接口
- ✅ 避免因接口失败导致的误登出

---

### 修复 2：增强应用启动时的状态恢复

**文件：** `src/main.ts`

#### 变更前：
```typescript
// ❌ 只监听状态变化，不主动恢复
userStore.$subscribe((mutation, state) => {
  if (state.token) {
    localStorage.setItem('auth_token', state.token)
  }
  if (state.userInfo) {
    localStorage.setItem('user_info', JSON.stringify(state.userInfo))
  }
})
```

#### 变更后：
```typescript
// ✅ 主动从 localStorage 恢复状态
const storedToken = localStorage.getItem('auth_token')
const storedUserInfo = localStorage.getItem('user_info')

if (storedToken && storedUserInfo && storedUserInfo !== 'undefined') {
  try {
    const userInfo = JSON.parse(storedUserInfo)
    
    // 直接设置 Store 状态
    userStore.token = storedToken
    userStore.userInfo = userInfo
    userStore.permissions = userInfo.permissions || []
    userStore.menus = userInfo.menus || []
    
    console.log('[Main] ✅ 从 localStorage 恢复用户状态')
  } catch (e) {
    console.warn('[Main] ⚠️ 解析用户信息失败:', e)
    localStorage.removeItem('auth_token')
    localStorage.removeItem('user_info')
  }
}

// 监听状态变化，同步到 localStorage
userStore.$subscribe((mutation, state) => {
  if (state.token) {
    localStorage.setItem('auth_token', state.token)
  } else {
    localStorage.removeItem('auth_token')
  }
  
  if (state.userInfo) {
    localStorage.setItem('user_info', JSON.stringify(state.userInfo))
  } else {
    localStorage.removeItem('user_info')
  }
})
```

**改进点：**
- ✅ 应用启动时立即恢复用户状态
- ✅ 避免页面刷新后状态丢失
- ✅ 添加详细的日志输出，便于调试

## 🐛 常见问题排查

### 问题 1：登录后仍然跳回登录页

**可能原因：**
- localStorage 中保存的数据格式不正确
- 后端返回的数据结构不符合预期

**排查步骤：**

1. 检查 localStorage 数据：
```javascript
console.log(localStorage.getItem('auth_token'));
console.log(localStorage.getItem('user_info'));
```

2. 检查用户信息格式：
```javascript
const userInfo = JSON.parse(localStorage.getItem('user_info'));
console.log('userInfo:', userInfo);
console.log('permissions:', userInfo?.permissions);
console.log('menus:', userInfo?.menus);
```

3. 检查后端响应：
```javascript
// 在 Network 标签中查看 /api/auth/login 的响应
// 确保包含以下字段：
{
  "token": "...",
  "userId": "...",
  "username": "...",
  "permissions": [...],
  "menus": [...]
}
```

---

### 问题 2：页面刷新后丢失状态

**可能原因：**
- main.ts 中的状态恢复逻辑未执行
- localStorage 数据被清除

**排查步骤：**

1. 检查 main.ts 日志：
```
[Main] 应用启动，检查用户状态...
[Main] Token: true/false
[Main] UserInfo: true/false
```

2. 如果显示 false，检查 localStorage：
```javascript
console.log(localStorage.getItem('auth_token'));
console.log(localStorage.getItem('user_info'));
```

3. 手动恢复状态：
```javascript
// 在控制台执行
const token = localStorage.getItem('auth_token');
const userInfo = JSON.parse(localStorage.getItem('user_info'));

import { useUserStore } from '@/stores/user';
const userStore = useUserStore();

userStore.token = token;
userStore.userInfo = userInfo;
userStore.permissions = userInfo.permissions || [];
userStore.menus = userInfo.menus || [];

// 刷新页面测试
location.reload();
```

---

### 问题 3：路由守卫无限循环

**症状：**
- 页面不断刷新
- 控制台重复输出路由守卫日志

**可能原因：**
- `next({ ...to, replace: true })` 导致重复触发守卫

**解决方案：**

检查路由守卫中是否有条件判断，确保不会无限循环：

```typescript
// 添加路由已存在的检查
if (userStore.menus && userStore.menus.length > 0) {
  const dynamicRoutes = addDynamicRoutes(userStore.menus);
  let addedCount = 0;
  dynamicRoutes.forEach(route => {
    if (!router.hasRoute(route.name!)) {
      router.addRoute(route);
      addedCount++;
    }
  });
  
  // 只有添加了新路由才重新导航
  if (addedCount > 0) {
    next({ ...to, replace: true });
    return;
  }
}
```
