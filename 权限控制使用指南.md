### 1. 登录获取权限


## 🎯 权限指令

### v-permission 指令

用于按钮级权限控制，根据权限码显示/隐藏元素。

#### 基本用法

```vue
<template>
  <!-- 单个权限 -->
  <button v-permission="'article:create'">发布文章</button>
  
  <!-- 多个权限（满足任一即可） -->
  <button v-permission="['article:edit', 'article:delete']">
    编辑或删除
  </button>
</template>
```

#### 工作原理

- 如果用户有所需权限，元素正常显示
- 如果没有权限，元素会被从 DOM 中移除
- 支持响应式更新（当权限状态变化时）

#### 示例场景

```vue
<template>
  <!-- 只有有创建权限的用户可见 -->
  <el-button 
    v-permission="'article:create'" 
    type="primary"
  >
    发布文章
  </el-button>
  
  <!-- 只有有删除权限的用户可见 -->
  <el-button 
    v-permission="'article:delete'" 
    type="danger"
  >
    删除文章
  </el-button>
  
  <!-- 需要多个权限之一 -->
  <el-button 
    v-permission="['article:audit', 'admin:audit']" 
    type="warning"
  >
    审核文章
  </el-button>
</template>
```

## 👑 角色指令

### v-role 指令

用于基于角色的控制，根据用户角色显示/隐藏元素。

#### 基本用法

```vue
<template>
  <!-- 单个角色 -->
  <div v-role="'Admin'">
    管理员专属内容
  </div>
  
  <!-- 多个角色（满足任一即可） -->
  <div v-role="['Admin', 'VIP']">
    管理员或 VIP 可见
  </div>
</template>
```

#### 示例场景

```vue
<template>
  <!-- 仅管理员可见 -->
  <el-button 
    v-role="'Admin'" 
    type="primary"
  >
    管理后台
  </el-button>
  
  <!-- 仅 VIP 用户可见 -->
  <el-tag v-role="'VIP'" type="warning">VIP 标识</el-tag>
  
  <!-- 管理员或超级管理员可见 -->
  <el-button 
    v-role="['Admin', 'SuperAdmin']"
  >
    高级管理
  </el-button>
</template>
```

## 🎨 组合式函数

### usePermission

提供权限检查的便捷方法。

```typescript
import { usePermission } from '@/composables/usePermission';

const {
  hasPermission,    // 检查权限
  hasRole,         // 检查角色
  isAdmin,         // 是否管理员
  isVIP,          // 是否 VIP
  userInfo,       // 用户信息
  permissions,    // 权限码列表
  roles          // 角色列表
} = usePermission();

// 使用示例
if (hasPermission('article:create')) {
  // 执行创建操作
}

if (hasRole('Admin')) {
  // 管理员操作
}

if (isAdmin.value) {
  // 管理员专属逻辑
}
```

### useMenu

提供菜单操作的便捷方法。

```typescript
import { useMenu } from '@/composables/usePermission';

const {
  mainMenus,      // 一级菜单
  getChildMenus,  // 获取子菜单
  findMenuByPath, // 根据路径查找菜单
  allMenus       // 所有菜单
} = useMenu();

// 使用示例
const firstLevelMenus = mainMenus.value;
const childMenus = getChildMenus('menu-id-1');
const targetMenu = findMenuByPath('/dashboard');
```

### useDashboard

提供仪表盘数据的便捷方法。

```typescript
import { useDashboard } from '@/composables/usePermission';

const {
  stats,          // 完整统计数据
  totalArticles,  // 文章总数
  totalComments,  // 评论总数
  totalViews,     // 总访问量
  loadStats       // 加载数据
} = useDashboard();

// 使用示例
onMounted(async () => {
  await loadStats();
  console.log('文章数:', totalArticles.value);
});
```

## ⚠️ 注意事项

### 1. 权限码命名规范

建议使用 `资源：操作` 的格式：

```typescript
'article:view'      // 查看文章
'article:create'    // 创建文章
'article:edit'      // 编辑文章
'article:delete'    // 删除文章
'comment:audit'     // 审核评论
'user:manage'       // 用户管理
```

### 2. 角色命名规范

使用英文驼峰或大写：

```typescript
'Admin'      // 管理员
'VIP'        // VIP 用户
'Editor'     // 编辑
'SuperAdmin' // 超级管理员
```

### 3. 性能优化

- 权限数据在登录时一次性加载，避免频繁请求
- 权限指令会缓存检查结果
- 使用 computed 缓存权限判断结果

### 4. 安全性

- 前端权限控制仅为用户体验优化，不能替代后端验证
- 所有敏感操作必须在后端进行权限验证
- Token 过期时会自动清除权限数据并跳转登录页
